Introduction

From Adaptive to Generative UI

The evolution from context-aware interfaces to AI-composed experiences — and why the future builds on both

The Two Paradigms

Two Stages of the Same Evolution

Both respond to context. The difference is how they respond — and generative builds on what adaptive started.

Adaptive UI
  • Selects from pre-built UI variations using rules and context signals
  • Understands who the user is, what they’re doing, where and when
  • Developers author every possible variation in advance
  • Predictable, testable, deterministic
Generative UI
  • LLMs create novel UI compositions at runtime that never existed before
  • Understands intent and generates the right interface on the fly
  • No pre-built variations — the model composes what’s needed
  • Flexible, creative, non-deterministic
Adaptive UI in Practice

Context-Aware, Rule-Driven

Adaptive UI reads user context and selects the right pre-built experience. The interface changes — but every variation was designed in advance.

Google Maps
Adapts routes, ETAs, and UI based on transport mode (driving, walking, transit), real-time traffic, location, and time of day. The navigation UI reshapes itself around context — but every layout was pre-designed.
Spotify
Surfaces different playlists by time of day, listening history, and activity. Morning Mix at 7am, Deep Focus during work hours, Release Radar on Fridays. Adaptive rules select from a library of curated content.
Netflix
Personalizes artwork, row ordering, and category labels per user. You and a friend see different thumbnail images for the same show — each chosen by algorithms from pre-generated variants.
iOS / Android
Suggested apps, notification summaries, and widget content adapt based on usage patterns, location, and time. The OS surfaces what’s relevant — always from components the platform team built.
Generative UI in Practice

LLM-Composed, Runtime-Created

Generative UI doesn’t select from options — it creates the interface. The LLM decides what components to render, what content to show, and how to structure the response.

Vercel AI SDK
Streams React components from the server. An LLM decides whether to render a chart, a table, a form, or a card — then generates and streams the actual JSX. The UI literally didn’t exist until the model created it.
ChatGPT Canvas
Generates structured outputs — editable documents, code with inline annotations, tables, and interactive controls — composed at runtime based on conversation context. No templates, no pre-built layouts.
GitHub Copilot Chat
Renders interactive code suggestions, diff views, terminal commands, and explanations. The model chooses the right UI pattern for each response — code block, step-by-step guide, or comparison table.
v0 by Vercel
Generates complete UI components from natural language prompts. Describe what you need and it produces working React/Tailwind code with a live preview — every output is unique to the request.
Tradeoffs

Strengths and Limitations

Neither approach is universally better. The choice depends on what you’re optimizing for.

Adaptive UI Strengths
  • Predictable — every variation is tested and designed
  • Performant — no LLM latency, pre-built components render instantly
  • Accessible — easier to ensure WCAG compliance across known variations
  • Cost-efficient — no per-request inference costs
Generative UI Strengths
  • Flexible — handles novel requests with no pre-built variation needed
  • Scalable — no need to author and maintain thousands of UI variations
  • Personalized — responses are tailored to exact user intent
  • Creative — can compose UI patterns developers didn’t anticipate

Adaptive UI trades flexibility for reliability. Generative UI trades predictability for expressiveness.

The Spectrum

The real future isn’t choosing one over the other.

Adaptive UI provides the reliable, tested foundation. Generative UI handles the long tail — novel interactions, personalized compositions, and experiences that can’t be pre-designed. The best products will blend both: adaptive where predictability matters, generative where flexibility wins.

Closing

The Future is Hybrid

Adaptive UI for the known. Generative UI for the unknown. The interfaces of tomorrow will know when to follow the rules — and when to break them.
